# BLACK BOOK — 재무·회계 실무자 커뮤니티

## Overview
BLACK BOOK은 기업신용평가사 **크레디뷰(CrediVue)**가 운영하는 재무·회계 실무자 커뮤니티입니다. 핵심 비유는 *"실무자의 장부(ledger)이자 지침서"* — 콘텐츠·문답·자료·일정이 매일 한 줄씩 기록되는 책.

이 패키지는 서비스의 주요 화면(IA 넘버링 기준)을 담은 **디자인 레퍼런스**입니다. 정보구조:

- **1. 홈** — 커뮤니티 대문(모듈형 대시보드)
- **2-1. 콘텐츠 메인** — 콘텐츠 목록(아티클 / 전문가 기고)
- **2-2. 콘텐츠 상세** — 아티클 읽기 화면
- **3. 실무 이야기** — 커뮤니티 문답 목록 (+ 문답 작성 모달)
- **3-1. 실무 이야기 상세** — 문답 스레드(질문 + 답변)
- **4. 자료실** — 실무 서식·템플릿 다운로드
- **5. 캘린더** — 정책자금·세무·회계·법령 일정
- **6. 파인더** — 정책자금 매칭 서비스 *사전 오픈 티저 + 알림 신청* (기능은 추후 확정)

> 홈 시안 탐색본(A 인덱스 / B 핀테크 / C 에디토리얼 / 시안 비교)은 이 패키지에서 **제외**했습니다. 채택된 THE LEDGER 방향만 포함합니다.

## About the Design Files
`designs/` 안의 `*.dc.html` 파일은 **HTML로 만든 디자인 레퍼런스(프로토타입)**입니다. 의도한 룩앤필과 동작을 보여주는 목적이며, 그대로 복사해 배포하는 프로덕션 코드가 아닙니다.

작업의 본질은 **이 디자인을 대상 코드베이스의 기존 환경(React/Vue/Next 등)과 패턴·라이브러리로 재구현**하는 것입니다. 아직 프론트엔드 환경이 없다면, 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

### 파일을 여는 법 / 구조 이해
각 파일은 사내 "Design Component(DC)" 런타임(`designs/support.js`) 위에서 동작합니다. 브라우저로 바로 열면 렌더됩니다. 구조는 두 부분입니다:

- **`<x-dc>` 템플릿** — 인라인 스타일로 작성된 마크업. `{{ ... }}`는 데이터 바인딩 홀, `<sc-for>`/`<sc-if>`는 반복/조건, `<span onClick="{{ handler }}">`는 이벤트.
- **`<script data-dc-script>` 로직** — `class Component extends DCLogic`의 `renderVals()`가 템플릿에 넣을 값(데이터/스타일/핸들러)을 반환. state와 setState로 상호작용 처리.

재구현 시 이 런타임 문법은 무시하고, `renderVals()`의 데이터 구조와 인라인 스타일 값만 참고하면 됩니다. **모든 스타일 수치·색·문구는 파일 내부에 그대로 들어 있습니다.**

## Fidelity
**High-fidelity (hifi).** 최종 색·타이포·간격·인터랙션이 확정된 목업입니다. 대상 코드베이스의 라이브러리로 **픽셀에 가깝게** 재현하세요. 단, `6. 파인더`는 서비스 구조가 미확정이라 *티저+알림신청* 단계까지만 hifi이며, 이후 검색/매칭 UI는 별도 정의 예정입니다.

### Typography
- **디스플레이/에디토리얼 헤드라인**: `Gowun Batang`(명조), weight 400/700. 클래스 `.bt`. 항상 `word-break: keep-all`.
- **UI·본문**: `Pretendard` (Pretendard Variable).
- **데이터·메타·아이브로우·수치**: `IBM Plex Mono`, `font-variant-numeric: tabular-nums`. 클래스 `.mono`.
- 스케일 예시: 홈/섹션 헤드라인 34–52px 명조 · 카드 제목 17–22px 명조 · 본문 14–17px · 메타/아이브로우 10–12px 모노(letter-spacing .06–.16em).
- 대비를 크게: 50px 명조 헤드라인 ↔ 10.5px 모노 라벨.

### Radius / Shadow / Border
- 모서리: 거의 직각. 태그·버튼·입력 **2~3px**. 큰 라운드 카드 금지(핀테크풍 지양).
- 그림자: 기본 **없음**(플랫 종이). 예외적으로 목록/상세의 흰 카드에 아주 옅은 `0 1px 2px rgba(24,27,30,.04), 0 5px 16px rgba(24,27,30,.05)`.
- 구분은 색 박스보다 **룰(hairline)**로. 섹션 헤더 상단은 1.5px 잉크 룰.

### Spacing / Layout
- 콘텐츠 max-width: **1180px**(GNB·푸터·대부분 화면), 리스트형 본문은 **1020px**, 읽기형 상세는 **760–800px**. 좌우 패딩 40px(상세 24px).
- 우측 고정 **fore-edge** 스트립: `width:22px`, `right:0`, `background:#EEF0F2`, 좌측 1px 보더, 세로 줄무늬 그라디언트. 다크 화면(파인더)에서는 `#17191C` 톤.
- 에디토리얼 비대칭 그리드 자주 사용(예: `1.6fr / .9fr`, `1.55fr / .95fr`).

## Screens / Views

### 1. 홈 (`1. 홈.dc.html`)
- **Purpose**: 커뮤니티 대문. 콘텐츠·문답·일정·자료를 한 화면에 요약.
- **Layout**: GNB → 본문 그리드 `1.62fr / .9fr`(좌 메인 모듈 / 우 사이드 모듈), max-width 1180.
- **주요 모듈**:

### 2-1. 콘텐츠 메인 (`2-1. 콘텐츠 메인.dc.html`)
- **Purpose**: 콘텐츠 목록. 두 발행 유형으로 구분.
- **Layout**: 헤더(INDEX 01 — 콘텐츠, 명조 타이틀, 소개 2문장) → 탭 → 콘텐츠 영역. max-width 1180.
- **탭(명조, 하단 블루 언더라인)**: `아티클 · 7` / `전문가 기고 · 5`.
  - **아티클**: 대표 히어로(좌 블루 차트 배너 16:9 / 우 텍스트, `아티클` 잉크 아웃라인 배지 + `세무 · 재무` 앰버 카테고리) → 카테고리 필터 칩(`전체/세무/회계/법령/재무/경영/정책자금`, 선택 시 잉크 배경 흰 글자) → 3열 카드 그리드(줄무늬 COVER 플레이스홀더 + 컬러 카테고리 + 명조 제목 + `크레디뷰 리서치 · N분 읽기 · 조회 …`).
  - **전문가 기고**: 이달의 기고 대형(4:5 포트레이트 + `VOL. 06 · 2025년 6월 · 전문가 기고`) → "지난 기고" VOL 원장 리스트(VOL/월 · 이니셜 아바타 · 명조 제목 · 전문가·직함).
- **Interaction**: 모든 콘텐츠 항목 클릭 → `2-2. 콘텐츠 상세`로 이동.

### 2-2. 콘텐츠 상세 (`2-2. 콘텐츠 상세.dc.html`)
- **Purpose**: 아티클 읽기.
- **Layout**: 읽기 컬럼 max-width **760px**(관련글 섹션만 1100px).
- **구성**: 브레드크럼 `← 콘텐츠 · 아티클` → 발행유형 배지(`자체 발행 아티클`, 잉크 아웃라인) + 카테고리(`세무 · 재무`) → 명조 42px 제목 → 리드문 → 저자 바(CR 아바타 + `크레디뷰 리서치` + `2025.06.11 · 8분 읽기 · 조회 1,240` + 저장/공유 아이콘) → 블루 히어로(차트 SVG) → 본문(`.abody` p 17px/1.85, h2 26px 명조, 블루 좌측 인용, 지표 카드) → 태그 → 저자 팔로우 카드 → 반응 바(도움돼요/댓글/저장) → "함께 읽으면 좋은 글" 3열.

### 3. 실무 이야기 (`3. 실무 이야기.dc.html`)
- **Purpose**: 커뮤니티 문답 목록. **채택 개념 없음 — 누구나 질문·답변**. 답변자에 운영사 크레디뷰(기업신용평가사) 포함.
- **Layout**: 헤더(INDEX 02, 타이틀, 소개, 우측 `＋ 문답 기입하기` 다크 버튼) → 그리드 **1020px** `1.6fr / .9fr`.
- **피드(좌)**: 정렬 탭 `요즘 뜨는 / 새 기록`(명조) + 건수 → 카테고리 칩(`전체/세무/회계/법령/실무/경영`) → 문답 행 리스트.
  - **문답 행**: 좌측 답변 수(모노 23px; 크레디뷰가 답하면 블루) + "답변" 라벨 / 카테고리(컬러 모노) + `크레디뷰 답변` 블루 배지(해당 시) + 시간(우측) / 명조 19px 제목 / 2줄 미리보기 / `작성자 · 조회 N · #태그`. 행 전체 클릭 → `3-1` 이동.
- **사이드(우)**: "오늘의 장부"(오늘 올라온 문답 38 · 누적 12,480 · 평균 첫 답변 42분 · 크레디뷰 답변 1,240) / "이달의 답변자"(최상단 고정: **크레디뷰 리서치** `운영사` 배지 · 기업신용평가사 · 답변 128, 이하 커뮤니티 답변자 랭킹, 답변 수 기준) / 다크 CTA 카드.
- **문답 작성(모달)**: `문답 기입하기`(헤더·사이드바) 클릭 → 중앙 모달. `background:rgba(15,17,19,.55)` 백드롭(클릭 시 닫힘, 내부 클릭은 stopPropagation) + 다이얼로그(max-width 620, `#F7F8FA`, 1px 잉크 보더, 상단 1.5px 잉크 룰 헤더 `INDEX 02 · 기입` + `문답 기입하기` + ×). 폼: 카테고리 칩(선택 시 카테고리색 배경) · 제목 input · 내용 textarea · 태그 input · `익명으로 기입` 체크 · `취소`/`문답 기입하기`.

### 3-1. 실무 이야기 상세 (`3-1. 실무 이야기 상세.dc.html`)

## Interactions & Behavior
- **네비게이션 이동**: 콘텐츠 항목 → 상세, 문답 행 → 스레드, 상세/모달의 `← 실무 이야기`·`취소` → 목록. (재구현 시 라우팅으로 연결)
- **탭/세그먼트 전환**: 콘텐츠(아티클/기고), 문답(요즘 뜨는/새 기록), 자료실(최신/인기) — state 기반 필터·정렬, 트랜지션 최소(페이드/0.2s).
- **필터 칩**: 카테고리 선택 시 리스트 필터링(선택 칩 잉크 배경 흰 글자).
- **문답 작성 모달**: 열기(버튼)/닫기(백드롭·×·취소), 카테고리 선택, 이메일/텍스트 입력. 백드롭 클릭 닫힘 + 다이얼로그 내부 stopPropagation.
- **파인더 알림 신청**: 이메일 정규식(`/.+@.+\..+/`) 충족 시 버튼 활성 → 클릭 시 완료 상태.
- **캘린더**: 카테고리 토글(표시/숨김), 월 이동, 셀/페이지 선택.
- **호버/프레스**: 링크 호버 → 블루(`--accent`), 프레스 → `--accent-press #16329E`. 큰 스케일/바운스 지양.
- **애니메이션**: 진입 모션 최소. `opacity:0` 시작 애니메이션 지양(콘텐츠가 숨지 않도록 기본 상태가 보이게).

## State Management
화면별 로컬 state로 충분(각 파일 `renderVals()` 참조):
- 콘텐츠 메인: `{ seg: 'article'|'contrib', filter }`
- 실무 이야기: `{ sort: 'hot'|'new', filter, composeOpen, ccat }`
- 자료실: `{ seg: 'new'|'hot', filter }`
- 캘린더: `{ off: {카테고리 숨김}, sel: 선택일, page }`
- 파인더: `{ email, done }`
- 데이터 페칭: 프로토타입은 하드코딩 배열. 프로덕션에서는 콘텐츠/문답/자료/일정/정책자금을 API로 대체(작성자·카테고리·유형·수치·날짜 필드 구조는 각 `_data()`/배열 참고).

## Copywriting & Tone
