# BLACK BOOK — THE LEDGER · Design System

> 재무·회계 실무자 커뮤니티 **BLACK BOOK**의 디자인 시스템.
> 운영사 **크레디뷰(CrediVue)** — 기업신용평가등급(공공기관용 등급확인서), AI 경영진단보고서, 기업모니터링, DB조회 플랫폼.

BLACK BOOK은 크레디뷰가 펴내는 커뮤니티입니다. 핵심 비유는 **"실무자의 장부(ledger)이자 지침서"** — 콘텐츠·자료·문답이 매일 *한 줄씩 기록*되는 책. 이 디자인 시스템은 그 비유를 시각 언어로 고정합니다: 흰색에 가까운 중성 종이, 잉크 괘선, 단 하나의 **블루** 액센트, 명조 헤드라인, 모노스페이스 수치, 폴더 인덱스 탭, 책 배지(fore-edge) 질감.

---

## CONTENT FUNDAMENTALS — 카피 & 톤

- **언어/호칭**: 한국어. 독자는 *실무자*(세무·회계·재무 담당, 대표). 격식 있되 현업 동료처럼. 과장·마케팅 호들갑 금지. 존댓말 기본("~합니다 / ~해요"), 헤드라인은 단정형("~법", "~다").
- **목소리**: 신뢰·차분·실무 중심. "장부에 기록한다", "한 줄씩 쌓인다", "오늘의 장부", "펼치다", "기입" 같은 *장부/책* 어휘를 의도적으로 사용.
- **헤드라인**: 명조(Gowun Batang), 짧고 단단하게. 예) "실무에서 막힐 때, 가장 먼저 펼치는 책.", "부채비율만 보던 여신 심사, 현금흐름까지 읽는 법".
- **메타/라벨**: 모노스페이스 + 영문 인덱스. 예) `INDEX 00 — 오늘 펼친 페이지`, `VOL. 06`, `EST. 2025 · CREDIVUE`, `매달 1회 · 이달의 기고`.
- **수치**: 항상 tabular-nums, 천단위 콤마. 예) `12,480`, `약 4.2억 원`, `142KB`.
- **카테고리 라벨**: 세무 / 회계 / 법령 / 실무 / 경영 / 재무 / 정책자금.
- **이모지**: 사용하지 않음. 강조는 색·서체·괘선으로.
- **CTA**: 동사 중심, 장부 은유 유지. "회원가입", "문답 기입하기", "사전예약", "정확한 진단 — AI 경영진단보고서", "전체 →".

## VISUAL FOUNDATIONS

**컨셉**: 인쇄된 장부 / 매거진 표지 / 서류철. 카드를 띄우지 않고 *종이 위에 직접* 콘텐츠를 얹고 잉크 괘선으로 구분한다.

- **컬러**: 흰색에 가까운 **중성 종이** 뉴트럴(`--paper-0..4`, #FFFFFF→#DCE0E4) + 쿨 잉크(`--ink-0..4`, #0F1113→#AEB6BE). 액센트는 **블루 단 하나** (`--blue` #2348D6, 시맨틱 `--accent`) — 활성 상태·핵심 수치·링크에만. 카테고리 색(파랑·앰버·그린·바이올렛)은 **콘텐츠 분류 태그 전용**, UI 크롬에 쓰지 않는다. 종이는 누렇지 않은 중성 톤.
- **타이포**: Gowun Batang(명조)=디스플레이/에디토리얼 헤드라인, Pretendard=UI·본문, IBM Plex Mono=데이터·메타·아이브로우. 큰 스케일 대비(50px 명조 헤드라인 ↔ 10.5px 모노 라벨)가 위계를 만든다.
- **배경/질감**: 단색을 피한다. (1) **그레인** 오버레이(SVG feTurbulence, multiply ~7.5%), (2) **장부 괘선**(36px 간격 가로 룰, `--ledger-rule`), (3) 오른쪽 고정 **fore-edge** 페이지 배지 스트립. 다크 섹션(스파인·파인더·푸터)에는 거대한 고스트 명조 숫자(opacity ~.04).
- **레이아웃**: max-width 1180 + 40px 패딩, 우측 22px fore-edge 여백. 에디토리얼 비대칭(1.55fr/0.9fr, 1.6fr/0.9fr). 콘텐츠는 *번호 매긴 장부의 줄*(01·02·03)로 나열. 섹션 헤더는 1.5px 잉크 룰 + 명조 h2 + 모노 메타.
- **보더/룰**: 1px 헤어라인(`--rule-hair` #D8D0BE)이 줄을 가른다. 섹션 상단은 1.5px 잉크 룰. 색 박스 테두리보다 *룰*로 구분.
- **모서리**: 거의 직각 — 태그·버튼·CTA 2~3px. 유일한 둥근 형태는 **폴더 인덱스 탭**(상단 9px). 핀테크풍 큰 라운드 카드 금지.
- **그림자**: 기본적으로 **없음**(플랫 종이). 떠 있는 단 하나의 유틸리티 = 대출 계산기 슬립에만 `--shadow-float`.
- **인덱스 탭(시그니처)**: 섹션 내비 = 파일 폴더/장부 인덱스 탭. 비활성 탭은 종이색(`--paper-3`)으로 뒤로 물리고, 활성 탭은 페이지 색으로 연결되며 상단에 블루 띠(inset 3px). 각 탭에 모노 번호(00·01…).
- **마스트헤드**: 신문 제호 구조 — 유틸리티 스파인(다크) → BLACK BOOK 제호(중앙, 양옆 VOL.·날짜 메타) → 인덱스 탭 → 표지 문구.
- **호버/프레스**: 호버 = 텍스트 잉크로 짙어짐 또는 블루로. 링크 호버 = `--accent`(블루). 프레스/액티브 = `--accent-press`. 큰 스케일·바운스 애니메이션은 지양, 절제된 페이드/0.2s 트랜지션.
- **애니메이션**: 진입 모션은 최소(필요 시 staggered rise). *백그라운드 iframe에서 콘텐츠가 숨지 않도록* opacity:0 시작 애니메이션은 피하고, 기본 상태가 보이게 한다.
- **이미지**: 실사 대신 **줄무늬 종이 플레이스홀더**(45° repeating-linear-gradient, `--paper-2`/`--paper-4`) + 모노 캡션(`PORTRAIT`, `COVER IMAGE`). 분위기는 차분하고 중성적.

## ICONOGRAPHY

- **스타일**: 얇은 라인 아이콘(stroke 1.8–2, 둥근 캡), 단색(currentColor). 면(fill) 아이콘은 거의 없음. 인라인 SVG로 사용(검색·화살표·문서·달력 등).
- **아이콘 폰트 없음**. 원본은 손으로 그린 인라인 SVG. 시스템 표준으로는 **Lucide**(CDN)를 권장 — 동일한 1.8~2 stroke·둥근 캡 라인 스타일과 가장 근접. *(대체 표기: 원본 커스텀 SVG → Lucide. 픽토그램 디테일이 중요하면 원본 SVG를 유지하세요.)*
- **유니코드 글리프**: 컨트롤에 사용 — 화살표 `‹ › ↗ →`, 계산기 접기 `— / +`. 모노스페이스로 렌더.
- **이모지**: 사용하지 않음.
- **로고/마크**: `assets/`에 잉크/화이트, 풀 워드마크/마크 단독 버전 보관(아래 INDEX 참고).

- `assets/` — 로고·마크:
  - `logo_ink.png` / `logo_white.png` — BLACK BOOK 풀 워드마크(마크+글자), 잉크/화이트.
  - `mark_ink.png` / `mark_white.png` — "B" 마크 단독.
- 레퍼런스 구현(홈 시안):
  - `1. 홈.dc.html` — **현재 채택 방향(THE LEDGER)**, 이 시스템의 기준. (구 `블랙북 홈 D 레저`)
